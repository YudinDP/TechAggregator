# TechAggregator 1.0 ("Ручная работа")

![Version](https://img.shields.io/badge/version-1.0.0-green.svg)
![Status](https://img.shields.io/badge/status-stable-brightgreen)

> TechAggregator - проект нацеленный на создание сервиса, позволяющего людям изучать и сравнивать разных товары сферы цифровой техники.   
> Ключевым аспектом проекта является отличительный упор на сравнение цен из разных популярных магазинов, маркеплейсов и прочих площадок с целью помочь пользователю выбрать самую подходящую технику с наибольшей выгодой, чего не предоставляют другие сервисы

## ✨ Особенности версии (что изменилось с последнего коммита)
- **[■■■■■■■■■□] 90%**
-  **Отлаженная фронтенд часть сайта**: Все страницы сайта и их функции работают исправно за исключением редактирования профиля(которое пока на этапе осмысления необходимости в этом) а также аналитика по сайту, то есть вывод статистики просмотров, заявок, отзывов и тд.
-  **База данных полностью готова**: На данном этапе туда более не планируется добавлять новых таблиц, полей и тд. Возможно появиться такая необходимость при переходе на сервер вместо локалки. А также всё еще бд недостаточно наполнена, там содержится 21 пример техники для теста функций в процессе разработки, но на релизе должно быть не менее 100, в том числе разных видов, например комплектующие ПК  
- **[■■■■■■□□□□] 60%**
-  **Серверная часть преимущественно налажена, но есть нюанс**: Ключевая проблема версии именно здесь, всё работает неплохо, кроме автоматических сборов данных, ни сама техника, ни опрос для истории цен не работают, работа над маршрутами и скриптами для них ведётся, но пока всё упирается в 401 ошибку и необходимость прокси для того чтобы обращаться к сайтам магазинов
-  **Вывод**: Для продолжения работы необходимо преобрести и подключить к запросам на сайты магазинов прокси, благодаря которому все запросы не будут падать на 401 оишбку. Или какое-то другое решение кроме прокси. Бесплатный прокси к слову не работает.  В остальной же части сайта необходимо подправлять мелкие недочёты и небольшие части недописанного функционала.  А также нужно для нормальной работы и избежания крупных проблем по работе сайта захостить его на сервер (по видимому VPS) уже в середине апреля. Такой срок выбран как раз для того чтобы за предстоящий месяц разобраться с доступом к сайтам магазинов (купить и настроить прокси).
